4. Feature-by-Feature Flow

Dashboard Flow

  • User visits /dashboard
  • getIndustryInsights() called
  • Check DB for IndustryInsight record
  • If missing → Gemini prompt → create record
  • Show salary chart (Recharts), demand level, key trends

Interview Quiz Flow

  • User visits /interview → click "Start Quiz"
  • generateQuiz() → Gemini → 10 MCQs JSON
  • User answers all 10 questions
  • saveQuizResult() → calc score
  • Wrong answers → Gemini improvement tip
  • Save Assessment to DB, show history chart

Cover Letter Flow

  • User visits /ai-cover-letter → fills form
  • jobTitle, companyName, jobDescription input
  • generateCoverLetter() → Gemini → Markdown letter
  • Saved to CoverLetter table in DB
  • View list of all cover letters
  • Delete or export as PDF (html2pdf.js)

Resume Builder Flow

  • User visits /resume
  • getResume() → load existing markdown content
  • Edit in markdown editor (@uiw/react-md-editor)
  • Click "AI Improve" → improveWithAI() → Gemini rewrites section
  • saveResume() → upsert in DB (one per user)
  • Export → html2pdf.js generates PDF

8. Key Design Patterns for Placement Interviews

Server Actions

  • No separate REST API layer needed
  • "use server" directive marks server function
  • Called directly from React components
  • Runs entirely on server = secure
  • No CORS issues, no fetch() needed

Route Groups

  • (auth) → centered login layout
  • (main) → protected + header layout
  • Parentheses DON'T affect URL path
  • Each group has own layout.jsx
  • Enables layout nesting in Next.js

Prisma Patterns

  • Singleton client in lib/prisma.js
  • Upsert for resume (one per user)
  • Transaction in updateUser()
  • Relations: User → Assessment, Resume, CoverLetter, IndustryInsight
